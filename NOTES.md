# Historical notes

`NOTES.md` is a historical documentation note, not a roadmap and not a product
feature catalog.

The previous version of this file duplicated README product vision text and kept
a long checked-off task ledger without evidence links. That content was removed
because it made the repository look more complete than the code and tests could
prove from the document itself.

Current source-of-truth split:

- Stable user-facing features and quickstart commands live in [`README.md`](README.md).
- Active and future work lives in [GitHub Issues](https://github.com/tomMoulard/atteler/issues)
  or linked project items so each item has an owner, discussion, and status.
- Detailed Symphony scheduler behavior lives in [`docs/symphony.md`](docs/symphony.md).
- Historical raw brainstorming remains available through git history rather than
  as a competing roadmap file.

When adding notes here, keep them dated, historical, and evidence-backed. Do not
add checked task lists or active roadmap items to this file.
