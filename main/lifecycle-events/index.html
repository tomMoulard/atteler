    Lifecycle hook events¶

    Lifecycle hook payloads use schema_version: 1 and are filtered through the selected payload policy before local hooks receive them.

    • metadata is the default telemetry-safe payload.
    • summary adds generated size/secret summaries for content and error fields.
    • full is for trusted hooks and still applies known-secret redaction plus size limits.

    agent_execute¶

    Emitted when a configured agent is selected for work.

    Schema version: 1

    metadata example¶

    {
  "metadata": {
    "agent": "reviewer"
  },
  "timestamp": "2026-01-02T03:04:05Z",
  "schema_version": 1,
  "event_id": "evt-doc-agent_execute",
  "type": "agent_execute",
  "session_id": "session-123",
  "agent": "reviewer",
  "model": "openai/gpt-4.1-mini",
  "payload_mode": "metadata",
  "redacted": true
}

    summary example¶

    {
  "metadata": {
    "agent": "reviewer"
  },
  "timestamp": "2026-01-02T03:04:05Z",
  "schema_version": 1,
  "event_id": "evt-doc-agent_execute",
  "type": "agent_execute",
  "session_id": "session-123",
  "agent": "reviewer",
  "model": "openai/gpt-4.1-mini",
  "payload_mode": "summary",
  "redacted": true
}

    full example¶

    {
  "metadata": {
    "agent": "reviewer"
  },
  "timestamp": "2026-01-02T03:04:05Z",
  "schema_version": 1,
  "event_id": "evt-doc-agent_execute",
  "type": "agent_execute",
  "session_id": "session-123",
  "session_path": "/home/alice/.atteler/sessions/session-123.json",
  "agent": "reviewer",
  "model": "openai/gpt-4.1-mini",
  "payload_mode": "full"
}
